## 1. 机器学习框架回顾

我们在构建一个机器学习模型时，本质上还是在进行那三个核心步骤：
1. 寻找一个带有未知参数的函数（模型）。
2. 从训练数据 (Training data) 中定义损失函数 (Loss)。
3. 进行优化 (Optimization) 寻找使得 Loss 最小的最佳参数 $\theta^*$。

但是，在实际操作中，如果模型效果不好，我们该如何定位问题？本节课提供了一份**万能的“模型排错与优化指南” (General Guide)**。

## 2. 模型诊断指南 (General Guide)

当训练好一个模型后，我们首先要观察的是**它在训练数据上的表现 (Loss on training data)**。

### 场景一：训练集上的 Loss 很大

如果连在训练集上，模型都无法将 Loss 降到足够低，这通常有两种可能：

1. **模型偏差 (Model Bias):** 
   * **现象：** 你的模型太简单了，这就像是“你想在海里捞针，但针根本不在海里”。不论你如何优化，模型本身就没有能力表达出正确的函数。
   * **解决办法：** 让模型变得更加复杂灵活。比如增加更多的特征 (Features)，或者使用深度学习增加神经元的数量和层数 (Neurons, Layers)。

2. **优化问题 (Optimization Issue):**
   * **现象：** 模型其实足够复杂了，但是你采用的优化算法（比如简单的梯度下降法）卡在了局部最优或者由于其他原因无法找到最佳参数。这就像是“针在海里，但你找不到它”。
   * **如何区分二者？** 李宏毅老师给出了一个巧妙的建议：**从一个比较浅的网络（或者简单的模型）开始。** 如果你把网络加深了，但在训练集上的 Loss 反而比浅层网络还要大，那说明复杂的模型没有被优化好，这就是 Optimization issue。
   * **解决办法：** 使用更强大的优化技术（如 Adam 优化器等，这在后续课程中会讲到）。

### 场景二：训练集上的 Loss 很小，但测试集上的 Loss 很大

如果在训练集上表现得非常好，但在测试数据 (Testing data) 上表现很差，这同样有两种可能：

1. **过拟合 (Overfitting):**
   * **现象：** 模型为了在训练集上取得极低的 Loss，变得“过于灵活”而开始在数据点之间“放飞自我 (freestyle)”。它死记硬背了训练数据，但没有学到真正的规律。
   * **解决办法：**
     * 增加训练数据 (More training data)；在无法增加真实数据的情况下，可以使用**数据增强 (Data augmentation)**，比如将图片翻转、裁剪。
     * **给模型增加约束 (Constrained model):** 减少特征、减少参数数量（比如 CNN 就是对全连接层做了参数共享约束）、提前停止训练 (Early stopping)、正则化 (Regularization)、Dropout 等。
     * **注意：** 约束不能过头，否则又会退化成 Model Bias 问题。

2. **数据不匹配 (Mismatch):**
   * **现象：** 训练数据和测试数据的分布根本不一样。比如，你用真实猫狗的照片训练模型，却让它去识别简笔画的猫狗。
   * **解决办法：** 这不仅仅是增加数据量能解决的，需要深入了解数据的生成背景和分布，使用 Domain Adaptation 等技术处理。

## 3. 偏差-复杂度权衡 (Bias-Complexity Trade-off)

随着模型复杂度的增加（特征变多、参数变多）：
* **训练误差 (Training loss)** 会一直下降。
* **测试误差 (Testing loss)** 则会呈现一个 **“U”型曲线**。在复杂度增加的初期，测试误差跟着下降；但一旦复杂度过了某个临界点，模型开始过拟合，测试误差反而会急剧上升。
* 我们的目标就是找到 U 型曲线底部对应的“最佳复杂度”。

---

## 4. 交叉验证与模型选择 (Cross Validation)

在 Kaggle 等竞赛或实际开发中，我们通常会有 Public Testing Set（能看到分数的测试集）和 Private Testing Set（最终排名的测试集）。

**绝对不要用 Public Testing Set 的结果来挑选你的模型！** 
如果你尝试了 100 个模型，选了一个在 Public Set 上表现最好的，这实际上等同于你“人工地”让模型在 Public Set 上过拟合了，它在 Private Set 上的表现大概率会非常糟糕。

### 正确的姿势：切分验证集 (Validation Set)
1. 将你手头的 Training Set，切分为两部分：真正的 **Training Set** 和用来挑选模型的 **Validation Set**。
2. 用 Training Set 训练多个候选模型。
3. 用 Validation Set 的分数来决定最终选用哪个模型。
4. 选好模型后，再把 Training Set 和 Validation Set 合并，用选定的模型结构在全量数据上再训练一遍，最后才拿去测试。

### 进阶：N 折交叉验证 (N-fold Cross Validation)
为了避免切分数据带来的随机性影响，你可以将数据切分成 $N$ 份（比如 3 份）。每次拿其中 1 份当 Validation，另外 2 份当 Training，这样一个模型就能训练出 3 个版本，算出 3 个 Validation 分数。最后取平均分数来评判哪个模型最优秀。

---

## 总结
遇到模型效果不好时，不要盲目试错，应该按照树状图进行排查：
1. **看训练集 Loss：** 降不下去就去排查 **Model Bias** 或 **Optimization Issue**。
2. **训练集 Loss 低了，看测试集 Loss：** 表现不佳就去排查 **Overfitting**（增加数据或加约束）或 **Mismatch**。
3. **选模型：** 永远使用切分出的 **Validation Set** 来做决定，而不是通过看最终测试结果来反复试错。
