# 第8讲 学习笔记：通用模型的终身学习 (Lifelong Learning for Foundation Models)

## 执行摘要 (Executive Summary)

本讲探讨了「通用模型的终身学习」，即在模型发布后，如何让其持续学习新技能、更新知识或遗忘不当资讯（Machine Unlearning），这个过程被称为**后训练 (Post-training)** 或 **持续学习 (Continual / Lifelong Learning)**。课程强调了后训练面临的核心挑战——**灾难性遗忘 (Catastrophic Forgetting)**，并明确指出后训练成功的评估应包含三个维度：修改的成功率 (Reliability)、举一反三的能力 (Generality)，以及不影响无关知识的局部性 (Locality)。

李宏毅教授指出，**「最好的后训练就是不要后训练」**，在动用后训练前应充分尝试提示工程 (Prompt Engineering)。若必须修改参数，本讲详细介绍了四种技术路径：**直接梯度下降微调 (Gradient Descent Fine-tuning)**、**模型编辑 (Model Editing, 如 ROME)**、**模型合并 (Model Merging)**，以及颠覆传统训练框架的 **测试时训练 (Test Time Training, TTT)**。

---

## 核心概念 (Key Concepts)

1. **后训练 (Post-training)**：针对来路不明（未公开原始训练资料）的开源基础模型 (Foundation Model) 进行参数更新，以适应特定任务。
2. **机器遗忘 (Machine Unlearning)**：更新参数的目的并非学习新知，而是让模型遗忘不当或有版权问题的资讯。
3. **后训练的三大评估指标**：
   - **Reliability (可靠性)**：确切达成修改的目标（如：将鲁夫的果实修改为尼卡果实）。
   - **Generality (泛化性)**：模型能举一反三（如：问「谁吃了尼卡果实」能回答「鲁夫」）。
   - **Locality (局部性)**：无关的知识不受影响（如：问「乔巴吃了什么」不应改变原本的答案）。
4. **灾难性遗忘 (Catastrophic Forgetting)**：微调过程中，模型在学习新技能的同时，大面积丧失过去已学会的能力（如失去 Alignment 防御力或输出格式控制力）。
5. **经验回放 (Experience Replay)**：将旧资料与新资料混合训练以防止遗忘。对于未公开资料的模型，可诱导模型「自问自答」生成旧资料来实现。
6. **任务向量 (Task Vector)**：微调后模型与基础模型的参数差值（$\Delta\theta$），代表了模型习得的特定技能或领域能力。

## 详细内容解析 (Detailed Breakdown)

### 1. 后训练的必要性与挑战
- **持续学习的需求**：
  - **知识更新**：如美国总统换人。
  - **学习新技能**：如让模型看懂注音符号或具备新的模态（听懂语音）。
  - **概念更新**：修正过时的观念。
  - **机器遗忘 (Unlearning)**：消除版权争议或移除训练时不小心记住的隐私个资。
- **核心难题：缺乏 Locality**
  - 使用常见的 Cross Entropy Loss 进行微调时，模型唯一的目标是降低 Loss，它可能会找捷径（例如：看到「谁是...」就一律回答某个答案），导致全域逻辑崩坏，也就是发生灾难性遗忘。

### 2. 技术一：直接使用梯度下降微调 (Gradient Descent Fine-tuning)
面对遗忘问题，可以从几个方向改进微调方法：
- **限缩参数更新范围 (LoRA)**：只更新少量参数。LoRA 可以减少遗忘 (Forgets less)，但也可能降低新知识的学习效果 (Learns less)，是双面刃，无法彻底根除遗忘。
- **加入正则化 (Regularization)**：加入人类对参数的偏好（$R$ 项），针对对基础模型能力重要的参数给予较大的 $\lambda_j$ 惩罚，限制其变动。
- **修改损失函数 (Loss Function)**：加入额外的对照训练资料。例如在教新知识的同时，混入无关的常识问答，强制模型保持 Locality。然而，这可能使模型变得过度简短（省话），因为训练集中的额外资料答案往往较短。
- **Experience Replay (经验回放)**：
  - **传统做法**：混合旧训练资料与新资料。
  - **创新解法**：面对缺乏原始训练资料的开源模型，利用语言模型的生成能力，让模型自己生成过去看过的问答（自问自答），再加入新知识一起训练。

### 3. 技术二：模型编辑 (Model Editing - ROME)
如果把梯度下降看作黑箱，模型编辑则是「人类主导的脑部手术」。
- **ROME (Rank-1 Model Editing)** 的逻辑：
  1. **定位知识储存位置**：透过遮蔽关键字并置换 Representation，找出对输出影响最关键的神经元。实验发现知识的检索通常发生在特定关键字最后一个 Token 的中间层（如第 18 层的 Feed Forward Network）。
  2. **直接修改参数**：计算出一个新的向量 $v^*$，并利用 Closed-form solution 修改权重 $W$ 成为 $W^*$。
  3. **维持 Locality**：在修改参数的公式中，加入限制条件，确保其他不相关的主体（如艾菲尔铁塔、金字塔）通过该网路层时，输出依然保持不变。

### 4. 技术三：模型合并 (Model Merging)
一种无需额外算力、直接将模型参数进行向量加减的神奇技术。
- **核心原理**：定义 **Task Vector (任务向量 $\tau$)** = 微调后模型 ($\theta_A$) - 基础模型 ($\theta$)。
- **模型相加**：可以将不同人训练的能力叠加。例如，将拥有「中文能力」的任务向量，加上拥有「安全防御 (Alignment)」的任务向量，加回基础模型，得到一个既会中文又有防御力的新模型。
- **模型相减 (Machine Unlearning)**：减去代表某种特定能力（如 PTT 乡民用语或特定词汇知识）的任务向量，可使模型回归「纯洁」，彻底忘记该知识。
- **类比推演**：若任务 A 类比于 C（合成语音 $\rightarrow$ 语音辨识），任务 B 类比于 D（真实语音 $\rightarrow$ 语音辨识），可利用 $\tau_A - \tau_B + \theta_C \rightarrow \theta_D$ 的参数运算，在没有目标领域真实语音资料的情况下，凭空拼凑出在目标领域真实语音上表现良好的模型（Syn-to-Real 的应用）。

### 5. 技术四：测试时训练 (Test Time Training, TTT)
打破「先训练，后测试」的常规，在 Inference 阶段即时更新参数。
- **运作机制**：当收到一笔测试资料 $x$ 时，不马上输出答案，而是当场微调模型参数，产生一个专为 $x$ 客制化的新模型，再输出答案。
- **如何无标签微调？**
  - **Semi-supervised Learning 概念**：使用测试资料当作无标签资料。例如 **TENT** 演算法利用 Entropy Minimization（最小化输出熵），让模型输出机率分布更集中。
  - **SUTA (Source-free Unsupervised Test-time Adaptation)**：在语音辨识中，仅需一句话即可当场微调模型并提升准确率。
  - **Pre-test Task**：利用自监督任务（如预测图片被旋转的角度）先微调模型，随后再进行真正的分类任务，能显著提升表现。
- **连续测试时训练 (Continuous TTT) 的挑战与解法**：
  - **挑战 (Model Collapse)**：如果每一笔新资料进来都累积更新参数，模型会因为不断的「灾难性遗忘」而生锈、崩坏，最终完全丧失能力。
  - **Dynamic SUTA (解法)**：结合 **Fast Update**（单笔资料更新，用完即抛，不累积）与 **Slow Update**（收集一批资料后再稳定更新，并永久保留），并加上自动的 Model Reset 机制，可有效在时间推移中持续提升模型表现。

## 结论 (Conclusions)

1. **后训练是把双面刃**：在追求 Reliability 的同时，极易破坏 Generality 和 Locality。在动用微调这项「手术」之前，应优先极致地发挥 Prompt Engineering（如 In-Context Learning）的潜力。
2. **评估的全面性至关重要**：不能仅以「目标问题答对了」来判定微调成功，必须严格测试模型是否丧失了原有能力或安全防线（Alignment）。
3. **参数运算展现了神经网路的数学奇迹**：Model Merging 证明了大型模型的参数可以在多个任务间直接进行线性加减，这为不共享资料的开源生态提供了一条极具潜力的协作路径。
4. **TTT 开启了模型动态适应的新局**：Test Time Training 赋予了模型在无标签的 Inference 阶段依然能即时进化的能力。虽然 Continuous TTT 还有 Model Collapse 的难题，但透过快慢更新分离的机制，已展现出人工智慧朝向「真正终身学习」迈进的曙光。
