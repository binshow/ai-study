# 机器学习 2025 笔记：第1讲 - 搞懂生成式人工智慧原理

## 执行摘要 (Executive Summary)
本堂课旨在介绍生成式人工智慧（如 ChatGPT、Gemini、Claude）的基础运作原理。课程分为理论解析与实作演练两大主轴。理论部分深入探讨了「语言模型」的本质，即一种透过「文字接龙（Auto-Regressive Generation）」来预测下一个 Token 的技术。实作部分则带领学习者认识开源模型（如 LLaMA）与非开源模型的差异，并使用 Hugging Face 平台进行单轮与多轮对话的实作演练，让学习者不仅知其然，更知其所以然。

## 核心概念 (Key Concepts)
1. **生成式人工智慧 (Generative AI)**：能够根据输入（Prompt）生成文字、影像或语音输出的人工智慧平台。
2. **语言模型 (Language Model)**：本质上是一个执行「文字接龙」的系统，透过未完成的句子预测下一个最合理的字词。
3. **Token**：生成式 AI 在进行文字接龙时的最小输出单位，不同于一般字典中的「代币」，它是模型在词汇表（Vocabulary）中可选择的选项。
4. **机率分布 (Probability Distribution)**：语言模型在生成每个 Token 时，会计算词汇表中所有可能选项的机率分数，并依此「掷骰子」来决定最终输出。
5. **自回归生成 (Auto-Regressive Generation)**：一种生成策略，将每次生成的 Token 附加回原来的 Prompt 后，再进行下一次的预测，直到生成「结束符号」。
6. **开源与非开源模型**：非开源模型（如 ChatGPT）仅提供 API 或网页介面；开源模型（如 LLaMA）则公开其内部参数与函数架构。

## 详细解析 (Detailed Breakdown)

### 1. 语言模型与文字接龙
生成式 AI 看似拥有高度智慧，能写 Email、找文法错误、写作业，但其最底层的运作逻辑非常简单——**文字接龙**。
当使用者输入一个问题（即 Prompt）时，模型会将这段文字视为一个「未完成的句子」，并试图猜测下一个合适的 Token 应该是什么。例如，输入「台湾最高的山是哪座」，模型会接续猜出「玉」，然后再猜出「山」，依序拼接出完整的答案。

### 2. 语言模型的输出机制：机率分布与掷骰子
模型并不是绝对确定下一个字是什么，而是计算出一个**机率分布**。
对于输入「人工」，模型会给予「智」50% 的机率、「呼」20% 的机率，而对毫不相关的符号（如英文字母或标点符号）给予极低的机率。接著，模型会根据这个机率分布进行「掷骰子」来选择输出的 Token。这些所有可能的 Token 集合被称为 **Vocabulary（词汇表）**。

### 3. 生成策略：自回归生成 (Auto-Regressive Generation)
语言模型的生成过程本质上是一个反复进行的**分类问题 (Classification Problem)**：
1. 接收一串输入符号 $Z_{1}$ 到 $Z_{t-1}$。
2. 在 Vocabulary 中选择下一个最可能的 Token $Z_{t}$。
3. 将 $Z_{t}$ 加回原输入序列中。
4. 重复上述步骤，直到模型选出代表「结束」的特殊 Token。

### 4. 开源与非开源模型
- **非开源模型**（Gemini, ChatGPT, Claude）：如同黑盒子，使用者只能给予输入并获得输出，无法得知其内部的数学函数 $F$ 或参数数值。
- **开源模型**（LLaMA, Mistral）：将模型的函数架构与参数数值公开，让开发者可以下载并自行部署研究。（注：部分开源模型并未公开其训练资料与过程，因此在严格定义下仍有争议。）
- **Hugging Face**：如同「模型的 Facebook」，是目前最大的开源 AI 模型集散地与资源库，提供开发者寻找与下载各类模型及资料集。

### 5. 实作：多轮对话的秘密
在实作演练中，讲者示范了如何透过程式码调用 LLaMA 模型。
单轮对话只需将 Prompt 传给模型；但若要实现类似 ChatGPT 的**多轮对话 (Multi-turn conversation)**，关键在于**必须将「历史纪录」连同「新输入」一起打包成一个新的 Prompt 丢给模型**。
因为语言模型本身没有记忆，每一次调用都是独立的文字接龙过程。只有将过去的对话历史完整保留并喂给模型，它才能够正确地根据上下文接续回答。

## 结论 (Conclusions)
第一讲为生成式人工智慧揭开了神秘的面纱，破除了大众对于 AI 「无所不知」的迷思。我们了解到强大的语言模型实际上只是在执行高度复杂的机率运算与文字接龙（自回归生成）。透过理解 Token、机率分布与多轮对话的历史纪录机制，我们不仅能更好地使用这些工具，更具备了进一步利用开源模型（如 Hugging Face 平台上的资源）进行应用开发的基础能力。接下来的课程将会建立在这个分类问题的基础上，探讨模型是如何被训练出来的。
